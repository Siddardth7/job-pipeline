# Scraper Pipeline Overhaul — Design Spec
**Date:** 2026-04-24
**Status:** Approved for implementation planning
**Audit basis:** `SCRAPER_AUDIT_REPORT_2026-04-19.md`
**Scope:** P0–P3 full sweep across all 10 audit findings (R1–R10)

---

## Overview

The scraper pipeline is operational but not trustworthy as an autonomous feed. Four categories of failure were verified: hidden stage failures masked by green workflow badges, inconsistent query coverage across sources, contaminated quality gates, and unbalanced source weighting. This spec defines the full fix across four implementation layers.

---

## Layer 1 — Observability (R1, R9, R10)

### 1.1 Workflow health gates
**File:** `.github/workflows/daily_scrape.yml`

Remove `continue-on-error: true` from three steps:
- Line 52: scraper orchestrator step
- Line 65: company intelligence step
- Line 103: distribute feed step

Add a `check_health` step at the end of the workflow:
- Reads the just-written `data/run_logs/YYYY-MM-DD.json` (not the old append-only `data/run_log.json`)
- Exits non-zero if: any critical source (ATS, JSearch, Apify, Adzuna) returned `status: error`, OR total `jobs_found` across all sources is 0
- USA Jobs and Contract scraper failures are warnings, not hard failures (new sources)
- A green workflow badge now means all critical stages completed cleanly

### 1.3 Metrics fix
**File:** `pipeline/company_intelligence.py`

`total_scraped` currently uses `total_input` (post-filter count). Fix:
- Orchestrator passes true raw count (sum of all source `jobs_found`) into `company_intelligence.py`
- `summary.total_scraped` reports that number (e.g., 503, not 79)
- Add `summary.total_post_filter` for the intermediate count
- Fixes the misleading metric gap (R9)

## Layer 2 — Query Alignment (R2, R7)

### 2.1 JSearch query-engine hookup
**File:** `scrapers/jsearch_scraper.py`

Current problem: Ignores `queries` parameter, uses hardcoded `TITLE_QUERIES` list.

Fix:
- Remove `TITLE_QUERIES` constant
- Accept `queries: List[str]` from orchestrator (generated by `query_engine.py`)
- Use those queries directly as search terms
- Quota alignment: Set `MONTHLY_LIMIT = 180` in scraper, set `JSEARCH_MONTHLY_LIMIT = 200` in orchestrator (20-query safety buffer = 180 effective). Both now agree.

### 2.2 Apify cluster coverage
**File:** `scrapers/apify_scraper.py`

Add missing clusters to `CLUSTER_TO_TITLE`:
```python
"industrial_operations": "Industrial Engineer OR Operations Engineer OR Supply Chain Engineer",
"mechanical_thermal":    "Mechanical Engineer OR Thermal Engineer OR Systems Engineer",
```

Fix `_extract_location`: Return `"Unknown"` instead of `""` when location is missing. Downstream tagging already handles `"Unknown"` via `location_confidence`.

### 2.3 SerpAPI → USA Jobs API
**New file:** `scrapers/usajobs_scraper.py`
**Remove:** `scrapers/serpapi_scraper.py`

USA Jobs (`data.usajobs.gov/api/Search`) is:
- Free — requires only a `User-Agent` header containing a contact email (not a secret, hardcoded in scraper)
- Structured JSON with clean location, salary, and posted date
- Highly relevant for aerospace/manufacturing target profile (NASA, DoE national labs, Army/Navy civilian manufacturing, DoD contractor roles)
- No rate limiting for reasonable daily usage (~20 queries/run)

Scraper design:
- Queries driven by `query_engine.py` cluster list (same contract as fixed JSearch and Apify)
- Fields mapped: `PositionTitle`, `OrganizationName`, `PositionLocationDisplay`, `ApplyURI`, `PublicationStartDate`
- Normalization outputs standard job shape matching existing pipeline schema
- Source tag: `usajobs`

Orchestrator changes:
- Replace `serpapi_scraper` import with `usajobs_scraper`
- Remove odd-day scheduling logic (USA Jobs runs daily)
- Remove `SERPAPI_*` quota constants from orchestrator
- Remove `serpapi` state tracking from `data/scraper_state.json`

Workflow changes:
- Remove SerpAPI environment variable (`SERPAPI_KEY`) from workflow secrets usage
- No new secrets needed (USA Jobs public API requires no auth)

## Layer 3 — Quality Gates (R3, R4, R5)

### 3.1 Company database re-baseline
**File:** `data/company_database.json`

Remove from GREEN tier and move to a new `staffing_reject` list:
- Actalent
- Jobot
- The Pivot Group Network
- The Pivot Group
- Motion Recruitment
- Myticas Consulting
- Jobs via Dice

**File:** `pipeline/company_intelligence.py`

Add `STAFFING_REJECT` constant loaded from the `staffing_reject` key in `company_database.json`. Any job from a company in this list is hard-dropped with `red_flag: staffing_reject` before company intelligence scoring runs. Same effect as F9 blacklist but semantically separated for staffing firms.

### 3.2 F7 relevance filter tightening
**File:** `pipeline/merge_pipeline.py`

Remove four overly generic tokens from `ROLE_RELEVANCE_TOKENS`:
```python
r"engineer i\b",   # matches any discipline
r"engineer ii\b",  # matches any discipline
r"engineer 1\b",   # matches any discipline
r"engineer 2\b",   # matches any discipline
```
The specific role-family tokens (manufacturing engineer, process engineer, quality engineer, etc.) already cover entry-level variants of target roles. These generic tokens were the primary path for off-target roles to survive F7.

### 3.3 F10 — Internship filter (new)
**File:** `pipeline/merge_pipeline.py`

New hard filter added after F7:
```python
INTERNSHIP_TOKENS = re.compile(
    r"\b(intern(ship)?|co[-\s]?op|coop)\b", re.IGNORECASE
)
```
Drops any job whose `job_title` matches. Runs as F10 in the filter sequence.

Weekly expected drop: ~5–15 jobs based on audit examples.

### 3.4 F11 — Location filter (new)
**File:** `pipeline/merge_pipeline.py`

New hard filter added after F10:
- Maintains a `NON_US_LOCATION_PATTERNS` list of recognizable non-US indicators: country names, country codes, and major non-US cities (e.g., `Abu Dhabi`, `Dubai`, `London`, `Toronto`, `Ontario`, `India`, `Germany`, `UK`, `UAE`, `Canada`, `Australia`)
- Drops jobs where `location` contains any non-US pattern (case-insensitive)
- **Blank locations pass through** — ATS jobs legitimately omit location; dropping them would remove the cleanest source. Already tagged `location_confidence: unknown` for downstream filtering if needed.
- Runs as F11 in the filter sequence.

## Layer 4 — Source Rebalancing (R6, R8)

### 4.1 Adzuna per-run cap
**File:** `scrapers/adzuna_scraper.py`

Add a `MAX_RAW_JOBS = 200` cap on Adzuna's total output per run. When the cap is reached:
- Stop fetching additional clusters
- Prioritize clusters in order of historical precision (core clusters first: manufacturing, quality, composites, materials, process; expansion clusters second: industrial_operations, mechanical_thermal, startup_manufacturing)
- Log how many jobs were dropped due to cap

Prevents single-source concentration events (R6 — the 2026-04-18 scenario where 163/222 final jobs came from Adzuna alone).

### 4.2 Contract scraper activation
**Files:** `engine/scraper_orchestrator.py`, `scrapers/contract_scraper.py`

Wire contract scraper into production flow:
- Add `from scrapers.contract_scraper import ContractScraper` to orchestrator imports
- Add orchestration block (same pattern as other scrapers): run daily, write `temp/jobs_contract.json`, record in `run_record`
- `pipeline/merge_pipeline.py` already whitelists `jobs_contract.json` — no change needed
- Add `contract` source to `check_health` step (warning-level, not hard failure)

### 4.3 Dead-code and README cleanup
**Files:** `README.md`, `pipeline/merge_pipeline.py`

- `README.md`: Remove all TheirStack references (scraper listing, architecture diagram, dependency table)
- `README.md`: Update source list to: ATS (Greenhouse + Lever), JSearch, Apify (LinkedIn), USA Jobs, Adzuna, Contract
- `README.md`: Update filter list to reflect actual F1–F11 stack
- `pipeline/merge_pipeline.py`: The `jobs_contract.json` whitelist entry stays (contract scraper now active)

## Success Criteria

| Criterion | Target |
|---|---|
| Workflow failure visibility | Any critical source error fails the workflow (no silent green) |
| Query cluster coverage | All 10 clusters covered by at least 2 sources |
| Non-US jobs in final feed | 0 |
| Internship jobs in final feed | 0 |
| Staffing agency jobs in final feed | 0 (hard-dropped, not just flagged) |
| Blank-location jobs | Tagged `location_confidence: unknown`, not silently mixed in |
| Adzuna single-source concentration | ≤ 200 raw jobs / run, ≤ 60% of final feed |
| Contract roles | Appearing in feed as distinct `employment_type: Contract` entries |
| `total_scraped` in summary | Reports true raw count, not post-filter count |
| Per-run audit logs | `data/run_logs/YYYY-MM-DD.json` present for every run |
